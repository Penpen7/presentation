# 作業方針メモ

ユーザーから指摘・フィードバックを受けたら、その都度このファイルに追記すること。

## プレゼン原稿・スライドの文体

- 太字（`**...**`）を使わない。強調は見出しやリストでつける。
- 「たったこれだけ」「実は」「なんと」のような煽り・カッコつけた表現を乱用しない。淡々と書く。使い所を決めずに散らすと浅ましく見えるので、効かせたい一箇所だけに絞る。

## スライドのデザイン（Marp / theme.css）

- フォントはヒラギノ（`Hiragino Sans`）を優先で使う。
- スライドのタイトル（`#` / h1）は上部に固定・中央揃え。コンテンツ量で上下しないようにする。
- タイトルは改行させない。1行に収まるよう短い文言にする。
- タイトルにカッコ書き（補足）は入れない。
- 2列（カラム）レイアウトは使わない。内容が分かれるときはスライドを分ける。
- スライドの本文は基本的に箇条書きにする（地の文や `##` 見出しの羅列で説明しない）。
- 箇条書きの文末は基本的に体言止めにせず、述語（動詞・形容詞など）で終える。
- スライド最下部に発表者の発言（`.talk` = アイコン`icon.svg` ＋ 吹き出し`.bubble`）を置ける。吹き出しはアイコンと同じ高さ・横幅いっぱい（右下のページ番号は避ける）。発言の長さで `bubble lg`（短い）/ `bubble`（標準）/ `bubble sm`（長い）を使い分ける。Marp は script を除去するため文字数での自動可変はできず、クラスで手動指定する。

## ツールの使い方

- Read など許可済みのツールは、いちいち確認せず勝手に使ってよい。
- ファイルの中身を見るときは `cat` / `head` / `sed` などのシェルコマンドではなく Read ツールを使う。
